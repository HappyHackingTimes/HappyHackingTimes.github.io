開発中、マウスを取り上げられたときに備えて覚えておきたいこと

  • 前置き
  • 覚えよ。さらば救われん
    • intelliJのプラグインの活用：Key Promoter X
    • ショートカットの候補をリスト表示：control + T
    • ショートカットをキーワード検索：command + shift + A
    • プロジェクトに関わるすべての検索：shift 2回連続で押す
  • 独断と偏見のショートカット集
    • 変数名を変更する：shift + F6
    • 選択文字列に合致する位置にカーソルをあてる：control + G
    • カーソルを指定方向に増やす：option 押したあとに option + ↑/↓
    • 新規ファイル作成：command + N
    • エクスプローラーにフォーカス/表示・非表示：command + 11
  • 変数呼び出し元に遷移：command + B
  • 変数呼び出し先を検索：option + F7
    • コードの折りたたみ/展開：command + shift + -/+
    • option + →/←で動く分だけ削除：option + delete
    • ファイル全体のコード整形：option + command + L
    • コードの移動：option + shift + ↑/↓
    • 関数の移動：command + shift + ↑/↓
    • タブの移動：command + shift + { / }
    • ウインドウの移動：option + (shift) + tab
    • 選択した対象の差分を表示：対象を選択して command + D
    • カンマ区切りの要素を交換：option + shift + command + →/←
    • 下の行を持ってきて一行にする：control + shift + J
  • コンテキストアクション
    • インターフェイスに準拠するメンバーの実装：option + enter
    • 存在しないクラス等の作成：option + enter

前置き

実プロジェクトではIntelliJをメインに開発を進めることになると思います。

基本操作はVSCodeなど他のIDEと変わることはありませんが、大きな利点として充実したショートカット機能が挙げられます。

ここではそのショートカットについて何を実際に使っているか？を使用シーンをGifで示しながら列挙します。
（随時更新します‥ﾀﾌﾞﾝ）

これらのショートカットを使いこなすことで開発スピードが上がること間違いなし！
また、VSCodeでも拡張機能があるみたい（IntelliJ IDEA Keybindings）です。
ぜひ活用してみてください。
